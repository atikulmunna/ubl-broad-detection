# v0.7.5 — 2026-03-01

## Fixed
- POSM aggregation crash after visit completion: `calculate_compliance` was setting `class_name` to a list when a planned item maps to multiple AI classes (e.g. PS PET SHEET items). The aggregator then passed the list to `get_posm_category()`, causing `TypeError: unhashable type: 'list'` and silently dropping the aggregated result. Fixed by always using `classes[0]` for `class_name`.
