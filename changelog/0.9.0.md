# v0.9.0 — 2026-03-08

## Added
- SOS two-stage pipeline: SOS-Detection.pt (1 class: product) → SOS-Classification.pt (47 brand classes) replacing single-stage DA_YOLO11X
- `_detect_products_two_stage_sos` in detection.py — bbox crop + chunked classification with OOM-safe halving
- `display_name` field in `sos_brand_shelving_norm.yaml` for proper brand name output (e.g. `gh` → "Glow & Handsome")

## Changed
- `analyze_share_of_shelf` now accepts `sub_category` param from client metadata; category comes from client, not model
- `sos_brand_shelving_norm.yaml` restructured to flat dict keyed by snake_case cls class names
- `config.yaml` share_of_shelf section: `conf` replaced with `det_conf`, `cls_conf`, `cls_batch_size`
- `MODEL_PATHS` in loader: `ubl` (DA_YOLO11X) replaced with `sos_det` + `sos_cls`
- Aggregator SOS processing updated for new flat brand norms and brand-level classification
- pipeline.py forwards `sub_category` metadata to `analyze_share_of_shelf`

## Fixed
- Aggregator `_load_brand_norms` updated to parse new flat YAML structure
- Brand norm lookup changed from `(brand, category)` tuple to flat `brand_key`
- `company_name` and `shelving_norm` in output now correctly resolved via BRAND_NORMS
