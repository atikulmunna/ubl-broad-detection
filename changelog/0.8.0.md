# v0.8.0 — 2026-03-01

## Added
- Vertical adjacency detection: when haircare shelftalkers are above/below skincare shelftalkers (no y-overlap between shelftalker centers), orientation is detected as vertical
- `vertical_adjacency_adjustments` in QPDS standards: haircare gets 4 legs, PONDS/GAL get 3 legs in vertical layout
- `tests/test_adjacency_detector.py` with 17 tests covering orientation, leg rules, and waiver logic
- `all_shelftalkers_detected` and `orientation` fields in adjacency output

## Changed
- `_detect_shelftalker_roi` now returns all detected shelftalkers (unfiltered) as third return value for orientation detection
- Shelftalker orientation uses center_y of all detected shelftalkers (not prefix-filtered) to avoid false negatives
- `shelf_talker_orientation_correct` outputs "Yes" when shelftalker is waived due to adjacency
